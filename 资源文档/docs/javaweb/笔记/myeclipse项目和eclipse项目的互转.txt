myeclipse项目和eclipse项目的互转

开发web项目可使用myeclipse或eclipse for java ee或eclipse for 企业平台(两种叫法一样，都是企业版的意思，不能用普通版eclipse，否则自己安装插件会很麻烦)，用于web开发的eclipse指的都是企业版eclipse。

两种工程项目的目录结构不同，生成class文件位置不同，但编程上没有区别。

myeclipse项目能自动配置web.xml配置文件，使用方便，但不是免费软件，商用开发更适合用eclipse。
eclipse是免费的，用法略复杂于myeclipse，编程时需要自己写web.xml，也可以不写使用默认配置（在运行时指定的server里有默认的web.xml），等发布时有需要再去写自己的web.xml(myeclipse项目的目录结构与tomcat基本一致，eclipse项目的目录结构不太一致，手动发布时也需要自己把相应文件拷贝到tomcat里对应位置)。

eclipse项目转myeclipse项目，新建一个myeclipse项目，在windows文件管理器里把java和jsp文件连带目录结构拖到myeclipse里相应位置即可。web.xml如有需要也要进行相应修改。

myeclipse项目转eclipse项目，也可以采用同样做法。


myeclipse和eclipse都可以通过菜单进入windows-preferences-server，对tomcat或运行环境进行配置，从而指定外部tomcat服务器作为运行环境。myeclipse相当于把编译后的项目拷贝到tomcat里；eclipse则使用所创建的server文件夹里的配置文件（server.xml,web.xml等）去启动tomcat服务器运行该web项目。

myeclipse项目可使用自带web服务器，也可以用外部tomcat运行，用外部tomcat时需要自己打开浏览器输地址，在运行同时就可以发布到tomcat里。eclipse项目不自带web服务器，只能用外部tomcat运行，需要双击console栏的server对发布进行配置才能在运行时发布到tomcat里，会引起tomcat全局配置的改变（myeclipse不会；eclipse发布使tomcat整个conf文件夹备份后产生新的conf文件夹），不建议使用eclipse进行发布。无论myeclipse还是eclipse项目都可以在tomcat里手动进行发布,推荐手动发布（在tomcat里建立目录结构，把web.xml,jsp文件和class文件拷到适当位置）。

注意myeclipse10里虽然可以选择javaee6，但在使用servlet3.0特性（如文件上传）时会出现一点问题。比如新建jee6的web项目后导入书上代码9.5的jsp和java，会发现import部分有org.apache.catalina.core.ApplicationPart找不到，此时可以把tomcat的lib目录下的catalina.jar包拖入项目的lib目录就可以解决报错问题（eclipse for jee就没有这个问题，有可能是myeclipse10版本不够才引起的）。运行的时候要选择后添加的tomcat7服务器（myeclipse10自带的服务器从启动提示上看是tomcat6，版本不够servlet3.0需求），此时打开浏览器输入地址会发现连index.jsp都不能打开，原因在于myeclipse会自动将一些Jar文件拷贝到tomcat相应web应用lib目录下，此时把该web应用Lib目录下的jar全部删除，关闭tomcat服务器，然后用startup.bat重启tomcat就可以在浏览器里正常运行index.jsp并成功实现文件上传（需有upload目录）。
经进一步测试只要删除catalina.jar然后用startup.bat重启tomcat就可以正常运行了，分析原因可能是在tomcat的lib下已经有此文件，如果在项目lib下再有就会在初始化时冲突导致连index.jsp都不能运行,必须把项目下的catalina.jar删去然后重启tomcat。缺少catalina.jar在myeclipse10下会报错，因此只能在myeclipse环境下引入该jar进行项目的编译发布，此时不能在myeclipse10环境下测试，只能停止服务器后删除该jar后再用tomcat里的startup.bat启动才能正常运行。(myeclipse只管编译发布，tomcat来运行)
